# 从一个黄点开始：用 ManimGL 做 32 种晶体学点群的对称操作动画

最近重新折腾晶体学点群的时候，又一次感受到了这套符号系统的微妙之处。

虽说晶体学点群只有 32 种，表格也并不算特别长，但真正想把每一个点群里的对称操作看明白，就会发现事情没有那么简单。旋转轴、镜面、反演中心、反轴、国际符号、Schoenflies 符号、矩阵表示、共轭类，这些东西单独看都能理解，一旦同时出现，就很容易变成一张“看起来都认识，但合起来不知道在干什么”的表。

所以这次的目标一开始很朴素：用 3Blue1Brown 版 Manim，也就是 ManimGL，给 32 种晶体学点群做一套可以看的 GIF。

后来这个目标一路扩展，逐渐变成了：不仅要有 GIF，还要有视频；不仅要展示几个代表操作，还要遍历整个点群；不仅要让物体动起来，还要把矩阵、点群符号、共轭类以及 proper/improper 操作的区别都放进画面里。

下面按实际推进顺序记录整个过程。为了避免把文章写成事后诸葛亮式的“最终方案说明书”，每一阶段都尽量标出两类内容：

- **我提出：** 观看效果后提出的新需求或修改意见。
- **此处为 AI 提出：** 为满足这个需求采取的技术处理或主动补充的方案。

这也比较接近这次工作的真实状态：不是一开始就把所有细节设计好了，而是一边生成视频，一边发现问题，再一边修改。

## 一、先把 ManimGL 跑起来

**我提出：**  
想用 `3b1b/manim` 做 32 种晶体学点群的 GIF。

**此处为 AI 提出：**  
第一步并不是写动画，而是先把环境打通。

这个项目至少需要几类工具：

- Python 虚拟环境，用来隔离 ManimGL 相关依赖。
- ManimGL，用来生成三维动画。
- FFmpeg，用来把视频转换成 GIF。
- LaTeX 工具链，用来显示矩阵、上划线和下标。
- 批量渲染脚本，用来一次性生成 32 个点群的结果。

ManimGL 本身负责把场景渲染成视频，FFmpeg 负责后处理。LaTeX 一开始看似只是锦上添花，后来证明它其实是必需品，因为晶体学点群符号里经常出现上划线和下标，例如 `\bar{4}`、`\mathrm{D}_{4h}` 这类写法，如果不用 LaTeX，很难排得自然。

这一阶段最麻烦的是 LaTeX 模板和本地 TeX 发行版之间的兼容问题。最后采用了较干净的中文 TeX 模板，让字幕、点群符号和矩阵都能稳定编译。

这一步解决之后，后面的工作才有了一个比较可靠的底座。

## 二、第一版：坐标轴、线框、一个黄点

**此处为 AI 提出：**  
环境跑通之后，先做了一个最小场景：

- 三维坐标轴。
- 晶体线框。
- 一个被操作的黄色小球。
- 点群标题。
- 当前操作字幕。

点群数据表中记录每个点群的基本信息，包括 Hermann-Mauguin 国际符号、Schoenflies 符号、所属晶系以及生成元。

对称操作统一写成矩阵，包括旋转、镜面反射、中心反演和反轴。这样一来，动画的核心问题就变成了：给定一个三维对象和一个 3x3 矩阵，如何把这个矩阵对应的几何操作表现出来。

第一版的目标很简单，只是验证“矩阵作用到三维对象上，动画能不能正常播放”。从这个角度看，一个黄点已经够用了。

当然，也只是“够用”而已。

## 三、黄点的问题：它太对称了

**我提出：**  
被操作的黄点对象最好换成一个低对称性的形状。

**此处为 AI 提出：**  
单个球最大的问题在于它本身太对称了。

对一个球做旋转，当然不容易看出区别；对一个球做镜面反射或中心反演，也几乎看不出变化。换句话说，操作确实发生了，但观众不一定看得出来。

于是把黄点改成了一个低对称性探针：

- 三个不同大小的小球。
- 黄、橙、红三种颜色。
- 两根连杆。
- 整体位置偏离原点。

这个改动看起来只是换了一个小模型，但对可视化非常关键。点群操作本身已经足够抽象，如果被操作对象还高度对称，观众很容易误以为“镜面反射没有发生”或者“中心反演没有效果”。

低对称性探针解决的正是这个问题。

## 四、背面看不清：透明度还不够

**我提出：**  
增加多面体透明度，因为小球转到视野后面之后看不清。

**此处为 AI 提出：**  
第一步是降低晶体线框的透明度，让背后的对象更容易看见。

**我提出：**  
还是不行，转到背面之后仍然看不到。

**此处为 AI 提出：**  
后来发现，单纯调透明度并不能彻底解决问题。三维渲染里还有深度测试，物体一旦被判定在后方，就可能被线框或其他对象遮挡。

于是进一步调整显示策略：

- 晶体线框保持半透明。
- 低对称性探针关闭深度测试。
- 探针始终作为教学对象优先显示。

这一步其实很有代表性。物理上真实的遮挡关系，和教学上希望观众看清楚的东西，并不总是一致。这里选择了教学优先：既然我们想看的是对称操作之后探针如何变化，就应该保证探针始终看得见。

## 五、先别急着做 32 个，拿几个点群试

**我提出：**  
先对其他几个示范点群进行尝试。

**此处为 AI 提出：**  
没有马上批量生成全部 32 个点群，而是先选了一组代表性点群做测试：

- `mmm`
- `-42m`
- `-3m`
- `6/mmm`
- `432`
- `m-3m`

这样可以覆盖正交、四方、三方、六方和立方晶系，也能覆盖旋转、镜面、反演、反轴以及高阶立方点群。

这一步的目的很朴素：不要等全部视频都生成完之后，才发现核心逻辑有问题。尤其是 `m-3m` 这种 48 元点群，如果它能正常渲染，说明很多复杂情况都已经被覆盖到了。

## 六、旋转还行，反射和反演不明显

**我提出：**  
旋转轴操作问题不大，但是镜面反射和中心反演之后对象没有明显变化。

**此处为 AI 提出：**  
这个问题有两个来源。

第一，早期的操作对象还不够低对称。第二，线框和探针一起变换时，观众有时不容易判断到底是对象变了，还是参照系变了。

因此后续明确了展示策略：

- 晶体线框可以参与变换，用来展示整体晶体对象的动态变化。
- 低对称性探针负责显示操作前后的差异。
- improper 操作额外用颜色变化强调。

这个阶段还没有正式加入颜色编码，但已经为后面区分 proper 和 improper 操作埋下了伏笔。

## 七、加入反轴操作

**我提出：**  
扩展到含有反轴的点群中试试看。

**此处为 AI 提出：**  
反轴可以理解为两步操作：

1. 绕某个轴旋转。
2. 再做中心反演。

如果用矩阵表示，可以写成：

```python
rotoinversion_matrix(axis, angle) = -rotation_matrix(axis, angle)
```

动画中则同时显示反轴方向和反演中心，让它和普通旋转轴区分开。这样一来，像 `-4`、`-42m`、`-3m`、`-6`、`-43m`、`m-3m` 这类点群就能更自然地纳入同一套框架。

## 八、从“代表操作”走向“完整点群”

**我提出：**  
想试试对于某个点群，把其中所有可能的对称操作遍历一遍。

**此处为 AI 提出：**  
这是整个项目的一个转折点。

最初的视频只是展示代表操作或生成元。例如，一个四重轴、一个镜面、一个反演中心。它适合说明“这个点群有哪些类型的对称元素”，但还不能说“遍历了这个点群”。

于是改成由生成元生成群闭包：

```python
operations = [identity]
seen = {identity}

while index < len(operations):
    current = operations[index]
    for generator in generators:
        candidate = generator @ current
        if candidate not in seen:
            operations.append(candidate)
            seen.add(candidate)
    index += 1
```

由于矩阵中存在浮点数，不能直接用矩阵相等来判断是否已经出现过。因此把矩阵四舍五入成稳定的 key，再用这个 key 去重。

随后检查每个点群生成出的阶数是否正确，例如：

```text
1       -> 1
-1      -> 2
mmm     -> 8
4/mmm   -> 16
6/mmm   -> 24
432     -> 24
m-3m    -> 48
```

这一步确认之后，视频才真正从“代表操作展示”变成“完整群操作遍历”。

## 九、线框到底要不要跟着动

**我提出：**  
在沿 `x=y` 面镜面对称时，长方体线框也发生了变化，这是否不合适？

**此处为 AI 提出：**  
这里涉及一个展示视角问题。

如果线框固定，它更像实验室坐标系；如果线框跟着动，它更像晶体对象本身接受了对称操作。前者适合强调“参照系不动，对象在变”；后者适合强调“这个晶体形状本身具有这样的对称操作”。

**我提出：**  
线框还是可以动，用来展示动态变化。

**此处为 AI 提出：**  
于是保留线框随操作变化的方案。

同时根据晶系调整参照线框：

- 三方、六方晶系使用六棱柱。
- 立方晶系使用立方体。
- 其他晶系使用低对称长方体。

这样线框既能作为对象的一部分参与变换，又不至于在不同晶系里显得过于随意。

## 十、逐步测试复杂点群

**我提出：**  
依次尝试 `-3m`、`6/mmm`、`432`，最后试最复杂的 `m-3m`。

**此处为 AI 提出：**  
这几个点群分别承担了不同测试任务：

- `-3m`：测试三方晶系和反演相关操作。
- `6/mmm`：测试六方晶系和多镜面操作。
- `432`：测试立方晶系中的 proper rotation 群。
- `m-3m`：测试 48 元完整立方点群。

`m-3m` 可以看作一次压力测试。只要它能稳定渲染，说明完整群遍历、字幕、矩阵、颜色变化和 GIF 转换基本都能承受复杂情况。

## 十一、给每个操作加矩阵

**我提出：**  
在字幕里把变换时对应的矩阵也输入一下。

**此处为 AI 提出：**  
右上角字幕加入当前操作的 3x3 矩阵。

早期矩阵用小数显示，比如 `0.50`、`0.87`。这种写法对程序友好，但对晶体学和群论的读者并不友好，因为它看起来更像数值计算结果，而不是一个清楚的数学表达式。

**我提出：**  
矩阵表示能否改成分数和根号的数学表达式？

**此处为 AI 提出：**  
把常见数值映射成 LaTeX 表达式，例如：

```text
\frac{1}{2}
\frac{\sqrt{3}}{2}
\frac{\sqrt{2}}{2}
\frac{1}{3}
\frac{2}{3}
```

三方、六方、立方点群中经常出现这些数。改成分数和根号之后，矩阵终于更像教材里的矩阵，而不是调试输出。

## 十二、右上角字幕和 LaTeX 标题

**我提出：**  
把操作字幕放到右上角。

**此处为 AI 提出：**  
画面布局逐渐固定下来：

```text
左上角：点群标题
右上角：当前操作、共轭类信息和矩阵
中间：晶体线框和低对称性探针
```

**我提出：**  
点群国际符号和 Schoenflies 符号都有上划线和下标，所以视频和 GIF 中都应该用 LaTeX 展示，并尽量避免文字重叠。

**此处为 AI 提出：**  
为 32 个点群建立 LaTeX 映射，例如：

```text
-4    -> \bar{4}
-43m  -> \bar{4}3\mathrm{m}
m-3m  -> \mathrm{m}\bar{3}\mathrm{m}
D4h   -> \mathrm{D}_{4h}
Td    -> \mathrm{T}_{d}
Oh    -> \mathrm{O}_{h}
```

标题拆成两行：第一行显示国际符号和 Schoenflies 符号，第二行显示晶系、完整群和元素数。这样既保留了晶体学符号的规范性，也避免左上角信息过于拥挤。

## 十三、批量生成 32 种点群

**我提出：**  
依次生成 32 种点群的视频和 GIF，命名格式为“晶系-点群国际符号”。

**此处为 AI 提出：**  
写批量渲染脚本，依次完成：

1. 调用 ManimGL 渲染每个点群。
2. 输出视频。
3. 用 FFmpeg 转换 GIF。
4. 按“晶系-国际符号”命名。

这里不列具体本机路径，因为路径对读者没有普遍性。公开记录中只需要保留“视频输出目录”和“GIF 输出目录”这类概念即可。

文件名还有一个小处理：国际符号中的 `/` 在文件名里容易被理解成路径分隔符，所以替换成全角符号，避免生成文件时出错。

## 十四、按共轭类排列，而不是按生成顺序排列

**我提出：**  
现在的视频已经包含所有操作，但希望按群中的不同 Class 进行排列。

**此处为 AI 提出：**  
这里的 Class 指群论中的共轭类，而不是简单地按“旋转、反射、反演”分类。

共轭关系可以写成：

```math
g \sim hgh^{-1}
```

对每个群元素计算它在群内的共轭轨道，就能把同一共轭类中的操作放到一起。

例如：

```text
432   -> [1, 3, 6, 8, 6]
m-3m  -> [1, 3, 6, 8, 6, 3, 6, 1, 6, 8]
```

右上角字幕也随之改成类似下面的形式：

```text
17/48  Class 4/10 elem 3/8
C3 rotation 120 deg
axis (1, 1, 1)
```

这样视频就不只是“把所有操作播放一遍”，而是开始显露出群结构本身。

## 十五、proper 和 improper 要一眼能分出来

**我提出：**  
希望体现 proper rotation 和 improper operation 的区别。可以让 improper 操作后的对象颜色发生变化，而 proper rotation 不变。

**此处为 AI 提出：**  
用矩阵行列式判断操作类型：

- `det = +1`：proper rotation。
- `det = -1`：improper operation。

视觉上采用颜色编码：

- proper rotation：探针保持黄、橙、红。
- improper operation：操作完成后，探针短暂变为蓝青、紫色，再恢复原色。

这样即使不仔细读右上角矩阵，观众也能感觉到“这类操作改变了取向性质”。

## 十六、字体太小，再统一字号

**我提出：**  
右上角有些字太小，希望放大一些，并统一字体大小。

**此处为 AI 提出：**  
问题来自长文本被整体压缩。于是做了几项调整：

1. 右上角操作说明统一字号。
2. 矩阵统一字号。
3. 放宽字幕区域宽度。
4. 长描述主动换行。

例如反轴操作不再挤成一行，而是拆成：

```text
48/48  Class 10/10 elem 8/8
rotoinversion bar 6: rotate 240 deg
axis (1, 1, 1), then invert
```

画面上的文字终于从“能塞进去”变成了“比较容易读”。

## 十七、最后得到的效果

最终每个点群动画包含：

- 完整群元素遍历。
- 按共轭类排序。
- LaTeX 国际符号。
- LaTeX Schoenflies 符号。
- LaTeX 矩阵。
- 低对称性探针。
- 与晶系对应的线框参照。
- proper/improper 颜色区分。
- 视频和 GIF 两种输出。

回头看，这套流程并不是一开始就设计完整的，而是一步步从问题里长出来的。比较关键的几次转折包括：

1. 从单个黄点改成低对称性探针。
2. 从代表操作改成完整群闭包。
3. 从生成顺序改成共轭类顺序。
4. 从普通文字改成 LaTeX 排版。
5. 用颜色区分 proper 和 improper 操作。

这些改动都不是为了让画面更“花”，而是为了让观众更容易判断：现在发生的到底是哪一种对称操作，它在群里处于什么位置，它对应的矩阵又是什么。

## 十八、一点体会

晶体学点群很适合用动画展示。

它一方面是抽象的群结构，另一方面又天然对应三维空间中的几何操作。只要把操作统一成矩阵，ManimGL 就可以同时展示几何动作、矩阵表达、点群阶数、共轭类结构以及 proper/improper 的差异。

相比只看点群表，动画能让很多“符号背后的几何意义”变得直观一些。尤其是反轴、镜面、中心反演和立方点群中的高阶操作，如果只在表格里看，很容易停留在记忆层面；一旦把对象真的转过去、映过去、反演过去，理解就会扎实很多。

后续还可以继续改进，例如：

- 给每个共轭类增加过渡页。
- 增加 proper/improper 图例。
- 使用更标准的晶体学操作记号。
- 输出更高清的版本。
- 继续优化三方和六方晶系的参照形状。

但到目前为止，这套流程已经完成了最初的目标：把 32 种晶体学点群做成一套可以直接观看的完整对称操作动画图谱。

此坑先算填上，后面如果继续补单形或者晶体定向，就又是另一个坑了。
